# biohacking research

Personal research notes to figure out what "biohacking" actually is,
which corners of it are substantive vs marketing noise, and whether
any of it is worth serious time.

## Why this exists

"Biohacking" is a load-bearing word that covers wildly different scenes:
DIY genetic engineering in garages, RFID chips in hands, medical device
security research, glucose monitors on healthy people, longevity clinics,
brain-computer interfaces, nootropic stacks, and Instagram wellness.

Reading a single article won't clarify which of these is which. This
repo is the accumulation layer: one file per subdomain, key figures and
orgs, primary sources, and a running fit-assessment.

## How to use

- Start with `00-what-biohacking-means.md` for the taxonomy.
- Each numbered file is a self-contained subdomain: what it is, who's
  in it, foundational sources, current state, red flags.
- `09-key-figures-and-orgs.md` is the cross-cutting people/org index.
- `10-reading-and-media.md` is the reading queue.
- `fit-assessment.md` is the running personal notes file — pros/cons,
  what's landing, what isn't, what to try next.

## Ground rules

- No shopping. This is research to *understand*, not to buy things.
- Distinguish primary sources (papers, project sites, community wikis)
  from secondary (journalism, YouTube, podcasts). Both are useful but
  the difference matters.
- Note publication dates. This scene has a decade+ of history and
  yesterday's revolutionary tech is often today's abandoned project.
- Flag anything that looks like influencer wellness with a "commerce"
  tag — not to dismiss it, but so it doesn't get mixed up with peer-
  reviewed work.
